Pelican

All these are... well, let's say not the easiest. Can update automatically with just the line afterwards. If it doesn't work, use make html. Apparently it works. ### Single instructions

To build HTML code (settings is optionnal)  
- `pelican /home/max/Studies_Work/6_website/content/ -s /home/max/Studies_Work/6_website/pelicanconf.py`
- `pelican content`

To build only certain content (dunno how this works yet):  
`pelican --write-selected output/posts/my-post-title.html`

To see the output from python 3 go to the _output_ folder and type:  
- `python -m http.server`  
- `pelican --listen`

### Continuous update
To update continuously both the building of the HTML version and the preview on the navigator, open two windows in the terminal.

- The first one is to check changes and build HTML continuously, go to the main folder and do either of the two following:
    - `pelican -r /home/max/Studies_Work/6_website/content/`
    - `pelican --autoreload /home/max/Studies_Work/6_website/content/`
- In the other tab (to update the navigator continuously)
    - go into the _output_ folder
    - `python -m http.server`

Note that the display may not be accurate if the _output_ folder is not emptied.

Ultimate command

To update changes made to files and update the navigator in a single command:
pelican -r /home/max/Studies_Work/6_website/content/ --listen
